# Contributing

Thank you for considering a contribution.

This repository is intentionally small and focused. Contributions are welcome, especially if they improve:

- clarity,
- reusability,
- AI-first onboarding,
- practical execution guidance,
- examples for real product teams.

## What kinds of contributions are helpful

Good contributions include:

1. improving the wording of the method,
2. adding better examples,
3. expanding the framework with reusable guidelines,
4. correcting contradictions or unclear sections,
5. adding templates that make AI-driven project setup easier,
6. improving discoverability, onboarding, and documentation flow.

## Before opening a PR

Please make sure your change:

1. keeps the framework simple,
2. does not add unnecessary ceremony,
3. preserves the principle that humans should not need to do manual setup work that AI can do,
4. stays consistent with the core method:
   - humans steer,
   - agents execute,
   - harnesses control quality.

## Preferred contribution flow

1. Open an issue if the change is large or changes the method definition.
2. For small fixes, open a PR directly.
3. Explain:
   - what changed,
   - why it matters,
   - whether it changes the method itself or just improves presentation.

## PR style

Please keep PRs:

- focused,
- small when possible,
- easy to review,
- explicit about trade-offs.

## Language policy

- Default repository docs should be in English.
- Localized versions can be added separately, such as `README.zh-CN.md`.

## PRs welcome

If you want to improve this framework, open a pull request.

Well-scoped PRs that improve clarity, usability, or adoption are especially welcome.
